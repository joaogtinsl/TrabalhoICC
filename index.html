O Git é um sistema de controle de versão distribuído amplamente utilizado para o rastreamento de alterações em projetos de software. Ele permite que você mantenha um histórico completo de todas as alterações feitas em seu código, facilita colaboração com outras pessoas e gerencie diferentes versões do seu projeto. Vamos explorar os comandos, Git mais comuns e como configurar e acessar um repositório no GitHub.

    • init: Este comando é usado para inicializar um novo repositório Git em um diretório local. Quando você executa git init, o Git cria uma pasta oculta chamada .git que contém todos os metadados necessários para o controle de versão.
    • clone: Utilizado para criar uma cópia local de um repositório Git remoto. Basta executar git clone URL_do_repositório para clonar um repositório.
    • branch: Permite criar, listar ou excluir branches (ramificações) em um repositório. Por exemplo, git branch lista todas as branches, e git branch nome_da_branch cria uma nova branch.
    • checkout: É usado para alternar entre branches ou restaurar arquivos. Por exemplo, git checkout nome_da_branch muda para a branch especificada.
    • status: Mostra o status atual do seu repositório. Indica quais arquivos foram modificados, adicionados ou removidos desde o último commit.
    • diff: Exibe as diferenças entre alterações feitas em arquivos. git diff mostra as diferenças entre o diretório de trabalho e o índice (staging area).
    • add: Adiciona mudanças ao índice (staging area) para prepará-las para um commit. Por exemplo, git add nome_do_arquivo adiciona um arquivo específico.
    • commit: Registra as alterações feitas no repositório. É acompanhado por uma mensagem que descreve as alterações. Por exemplo, git commit -m "Mensagem do commit".
    • push: Envia commits locais para um repositório remoto. Por exemplo, git push origin nome_da_branch envia os commits para a branch especificada no repositório remoto.
    • pull: Atualiza o repositório local com as alterações do repositório remoto. git pull combina as alterações do repositório remoto com o branch local atual.
    • revert: Desfaz um commit específico, criando um novo commit com as alterações revertidas.
    • merge: Combina alterações de diferentes branches. Por exemplo, git merge nome_da_branch mescla as alterações da branch especificada na branch atual.
    • stabh: Guarda temporariamente mudanças que não estão prontas para serem commitadas, permitindo que você limpe o diretório de trabalho.
    • rm: Remove arquivos do diretório de trabalho e do índice. Por exemplo, git rm nome_do_arquivo remove o arquivo especificado.
    • config: Configura opções específicas do usuário, como nome de usuário e endereço de e-mail. git config --global user.name "Seu Nome".
    • reset: Permite desfazer alterações no diretório de trabalho ou no índice. Por exemplo, git reset HEAD nome_do_arquivo remove o arquivo do índice.
    • remote: Gerencia repositórios remotos. Por exemplo, git remote add origin URL_do_repositório adiciona um repositório remoto chamado 'origin'.
    • fetch: Baixa commits, arquivos e referências de um repositório remoto para o repositório local. Por exemplo, git fetch origin atualiza as informações do repositório remoto 'origin'.
    • show: Exibe informações sobre commits, tags ou objetos Git. Por exemplo, git show hash_do_commit mostra informações sobre um commit específico.
    • help: Exibe a documentação de um comando específico. Por exemplo, git help commit mostra informações sobre o comando git commit.
    • rebabe: Aplica commits de uma branch em cima de outra branch. Por exemplo, git rebase branch_destino move os commits da branch atual para a branch destino.
    • blame: Mostra quem modificou cada linha de um arquivo e em qual commit essa modificação foi feita.
    • tag: Cria, lista ou verifica tags anotadas ou não anotadas. Por exemplo, git tag -a v1.0 -m "Versão 1.0" cria uma tag anotada com a mensagem especificada.

Os 5 mais usados

Nome Funcionalidade
init Prepara um diretório para controle de versão do Git.
clone Cria uma cópia local de um repositório remoto do Git.
add Adiciona mudanças ao próximo commit no repositório Git.
commit Registra as mudanças no repositório Git com uma mensagem descritiva.
